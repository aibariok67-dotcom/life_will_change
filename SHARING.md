# Как поделиться проектом с другом на Windows 🎮⚙️

Поскольку у вашего друга Windows и он не хочет работать с консолью и устанавливать Python, лучшим решением будет упаковать этот скрипт в **готовый запускаемый файл (.exe)**. 

После этого другу останется только дважды кликнуть по файлу, чтобы запустить камеру и трек!

Ниже описаны два основных способа реализации:

---

## Способ 1: Сборка .exe через PyInstaller (Самый надежный)

Так как вы работаете на Linux (Fedora), вы не можете напрямую собрать `.exe` для Windows (PyInstaller собирает программу только под ту ОС, на которой запущен). 

Но вы или ваш друг можете собрать его на любом компьютере с Windows, выполнив несколько простых шагов.

### Инструкция по сборке на Windows:
1. Установите Python на компьютер с Windows (обязательно выберите галочку **"Add Python to PATH"** при установке).
2. Скачайте/перенесите папку проекта `v_gesture_player` на этот компьютер.
3. Откройте командную строку в папке проекта и выполните установку библиотек и PyInstaller:
   ```cmd
   pip install -r requirements.txt
   pip install pyinstaller
   ```
4. Соберите проект в один файл с помощью следующей команды:
   ```cmd
   pyinstaller --onefile --noconsole --collect-all mediapipe main.py
   ```
   *Параметры:*
   * `--onefile` — собирает всё в один единственный файл `.exe`.
   * `--noconsole` — скрывает черное окно консоли при запуске (останется только окно вашей камеры).
   * `--collect-all mediapipe` — принудительно копирует все внутренние файлы MediaPipe (модели, ресурсы), без этого скрипт выдаст ошибку на старте.

5. Готовый файл появится в созданной папке **`dist/main.exe`**. Этот файл можно отправить другу через Telegram, Discord или флешку!

## Способ 2: Автоматическая сборка через GitHub Actions (Для профи)

Если вы зальете код в свой репозиторий на GitHub, вы можете настроить бесплатную автоматическую сборку (CI/CD). Каждая ваша отправка кода на GitHub будет автоматически запускать Windows-сервер в облаке, собирать `.exe` и выкладывать его в раздел "Releases" (Релизы).

Для этого просто создайте файл по пути `.github/workflows/build.yml` с таким содержимым:

```yaml
name: Build Windows EXE

on:
  push:
    branches: [ main ]

jobs:
  build:
    runs-on: windows-latest
    steps:
    - uses: actions/checkout@v4
    
    - name: Set up Python
      uses: actions/setup-python@v5
      with:
        python-version: '3.12'
        
    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt
        pip install pyinstaller
        
    - name: Build with PyInstaller
      run: |
        pyinstaller --onefile --noconsole --collect-all mediapipe main.py
        
    - name: Upload Release Asset
      uses: actions/upload-artifact@v4
      with:
        name: Persona5-GestureTrigger-Windows
        path: dist/main.exe
```
Ваш друг сможет просто зайти на страницу вашего репозитория и скачать собранный архив в один клик!

## Альтернатива: Веб-версия (HTML/JS в браузере)

MediaPipe имеет JavaScript-версию, которую можно запустить прямо в браузере. 
Однако современные браузеры из соображений безопасности **блокируют доступ к веб-камере** для обычных локальных файлов (при открытии через `file:///...`). 
Чтобы камера работала, страницу нужно запустить на сервере (например, `http://localhost:8000` или разместить в интернете на бесплатном хостинге вроде **GitHub Pages**, **Vercel** или **Netlify**).

Если вам интересен этот вариант — мы можем легко написать красивую веб-страницу, которую вы загрузите на GitHub Pages, и тогда другу достаточно будет просто перейти по ссылке в браузере!
